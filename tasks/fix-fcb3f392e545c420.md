# fix-fcb3f392e545c420 — E1T2: invite link sent but code not persisted when Redis unavailable

**Weight:** 0.0000 (share of project budget)
**Reward:** 0 SBOT

When `getRedisClient()` returns `null` (no `REDIS_URL`), the handler still calls `ctx.reply()` with the generated invite link. The generated `invite:<code>` key is never written to any persistent store, so the invite is unusable without any warning to the sender. A missing-Redis guard ("Could not create invite. Please try again later.") would prevent silently broken invites.
